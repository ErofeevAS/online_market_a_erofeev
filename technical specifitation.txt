	Техническое задание.
	Заголовок страницы: все страницы, кроме login
На страницах необходимо отобразить header и footer магазина. В верхнем правом углу должен отображаться email(username) аутентифицированного пользователя, по нажатию на которого происходит переход в профиль пользователя. Рядом должна располагаться кнопка logout. 
Слева должна располагаться панель навигации с кнопками навигации(для каждой роли свои):
    • Для пользователя с ролью Seller должны быть: Items, Orders, contacts, about us. 
    • Для пользователя с ролью Customer должны быть: Items, MyOrders, contacts, abouts us. 
    • Для пользователя с ролью Admin должны быть: Users, Articles.
    • Для пользователя с ролью SuperAdmin должны быть: Users, Articles. 

	Заголовок страницы: Items для Seller
По центру экрана разместить список Item. Должны быть следующие полями: Порядковый номер, name, price, description(укороченное), на удаление(checkbox), кол-во отзывов(показать отзывы). Снизу должны быть предусмотрены:
    • пагинация и возможность выбора количества отображаемых Item(10,20,50,100).
    • кнопка добавления Item(возможно имеет смысл вынести в nav-panel).
    • кнопка на upload Items с XML файла(если при обновлении возникают конфликтные ситуации, предложить перезаписать или пропустить);
    • кнопка импорта Items с базы данных в XML или JSON(возможно имеет смысл вынести в nav-panel)
    • кнопка удаления, которая удаляет все выбранные Items в столбце «на удаление».
Над списком разместить окно фильтрации, при вводе в которое отображаются Item с частичным совпадением с Item.name. Для price предусмотреть возможность установки максимально и минимальной цены. Также в столбцах name и price должна быть предусмотрена сортировка по возрастанию и убыванию. 
Заголовок страницы: Orders для Seller
 По центру экрана разместить список Order. Должны быть следующие поля: Порядковый номер, item.name, amount, totalprice, created, status(dropped list(NEW, REVIEWING, IN_PROGRESS, DELIVERED)), изменить статус. Снизу должна быть предусмотрена пагинация и возможность выбора количества отображаемых Order(10,20,50,100).
Над списком разместить окно фильтрации, при вводе в которое отображаются Orders с частичным совпадением поля Item.name. Для totalprice предусмотреть возможность установки максимально и минимальной цены. Также в столбце status предусмотреть фильтрацию по статусу. Предусмотреть checkBox notShowDelivered.

	Заголовок страницы: export для Seller

	Заголовок страницы: Item для Seller

	Заголовок страницы: Articles для Seller
Содержит список Articles с возможностью добавления, изменения и удаления комментария к существующим Article. Снизу должна быть предусмотрена пагинация и возможность выбора количества отображаемых Articles(10,20,50,100).

	Заголовок страницы: Articles для Customer
Содержит список Articles с возможностью создания нового Article и возможностью  добавления, изменения и удаления комментария к существующим Article. Снизу должна быть предусмотрена пагинация и возможность выбора количества отображаемых Articles(10,20,50,100).

	Заголовок страницы: Items для Customer
По центру экрана разместить список Item. Должны быть следующие поля: Порядковый номер, name, price, description(укороченное), кол-во отзывов(показать отзывы). Снизу должны быть предусмотрены:

Над списком разместить окно фильтрации, при вводе в которое отображаются Item с частичным совпадением с Item.name. Для price предусмотреть возможность установки максимально и минимальной цены. Также в столбцах name и price должна быть предусмотрена сортировка по возрастанию и убыванию.

	Заголовок страницы: MyOrders для Customer
По центру экрана разместить список Order для текущего пользователья. Должны быть следующие поля: Порядковый номер, item.name, amount, totalprice, created, status. Снизу должна быть предусмотрена пагинация и возможность выбора количества отображаемых Order(10,20,50,100).

	Заголовок страницы: Profile для Customer
Должен содержать информацию о пользователе(firstName, lastName, telephone, address, email) с возможностью изменения данных кроме(email). Кнопку сменить пароль вызывающая стандартную форму смены пароля.

На всех страницах c формами ввода предусмотреть валидацию входных параметров.
